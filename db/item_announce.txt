// 物品掉落公告配置文件
// 
// rAthena自帶的 conf/battle/drops.conf 中的 rare_drop_announce 選項是
// 根據物品掉率來判定是否需要進行公告的。
// 
// 然而有時候，我們希望只要是特定的道具，不管是什麼掉率，一旦掉落就公告。
// 如果有這種需求的話，可以使用本配置文件來解決問題。
//
// 數據庫格式說明:
// 物品編號,公告的觸發時機
//
// 公告的觸發時機:
//	&1: 打死魔物，魔物將道具掉落到地上時公告
//	&2: 打死魔物，掉落MVP獎勵到玩家的背包時公告
//	&4: 使用「偷竊」技能，從魔物身上偷到物品時公告
//
// 例如:
// 909,1   // 當打死魔物，魔物將傑勒比結晶掉落在地上時公告
// 713,3   // 當打死魔物，魔物將空瓶掉落在地上或當做MVP獎勵直接放到玩家背包時公告
//
// 注意:
// 您會發現「開箱子時，獲得了物品並不會公告」，這是因為在 rAthena 的官方設計中，
// 用來開箱子的 getgroupitem 指令擁有它自己的一套公告體系，具體請見 doc/item_group.txt
//
// 此外，您還可以在 db/[pre-]re/item_flag.txt 中為某個物品配置flag包含16，
// 這樣當此物品掉落在地面後，拾取到背包時(或開箱子直接開到背包中時)將自動進行全服公告。
